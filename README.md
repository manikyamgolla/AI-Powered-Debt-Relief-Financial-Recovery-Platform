# AI-Powered Debt Relief & Financial Recovery Platform

A full-stack web application that helps borrowers manage loans, understand their
financial health, and use AI to generate debt settlement recommendations and
negotiation letters.

## Tech Stack

**Frontend:** React (Vite) · Tailwind CSS · React Router · Axios · React Hook Form · Recharts
**Backend:** FastAPI (Python 3.12+) · SQLModel (SQLAlchemy + Pydantic) · SQLite (Postgres-ready)
**Auth:** JWT (python-jose) + password hashing (passlib, pbkdf2_sha256)
**AI:** Google Gemini (`google-genai`), with deterministic fallback logic if the API is unavailable
**DevOps:** Docker, docker-compose, GitHub Actions CI

## API Endpoints

| Method | Path | Description |
|---|---|---|
| POST | `/auth/register` | Create a user |
| POST | `/auth/login` | Get a JWT access token |
| GET | `/auth/profile` | Get current user (JWT required) |
| POST | `/loan` | Create a loan |
| GET | `/loan` | List the user's loans |
| GET | `/loan/{id}` | Get a loan |
| PUT | `/loan/{id}` | Update a loan |
| DELETE | `/loan/{id}` | Delete a loan |
| POST | `/financial-analysis` | Compute surplus, EMI/DTI ratios, stress level, health score |
| GET | `/dashboard` | Dashboard summary |
| GET | `/analytics` | Chart-ready data (loan distribution, EMI series) |
| POST | `/settlement` | Generate an AI settlement recommendation for a loan |
| GET | `/settlement/{id}` | Retrieve a stored recommendation |
| POST | `/generate-letter` | Generate an AI negotiation letter |
| GET | `/letters/{id}` | Retrieve a saved letter |
| GET | `/history` | List all AI prompt/response history |
| GET | `/history/{id}` | Get a single history entry |

Full interactive documentation is auto-generated by FastAPI at `/docs` (Swagger UI)
and `/redoc` once the backend is running.

## Database Schema

- **users**(id, name, email *unique*, hashed_password, created_at)
- **loans**(id, user_id → users, lender_name, loan_type, outstanding_amount, emi, interest_rate, overdue_days, tenure_months, status, created_at, updated_at)
- **financial_profiles**(id, user_id → users *unique*, monthly_income, monthly_expenses, savings, monthly_surplus, updated_at)
- **settlement_recommendations**(id, loan_id → loans, user_id → users, recommended_percentage, suggested_amount, monthly_payment_plan, ai_summary, created_at)
- **negotiation_letters**(id, loan_id → loans, user_id → users, letter_type, letter_text, created_at)
- **ai_history**(id, user_id → users, module_type, prompt_text, response_text, used_fallback, created_at)

An ER diagram can be generated from these models with a tool like `eralchemy` or `dbdiagram.io`;
paste the schema above or export from SQLite to visualize it.

## AI Logic

**Settlement recommendations** (`app/services/ai_service.py::generate_settlement_recommendation`):
sends income, expenses, outstanding balance, EMI, and overdue days to Gemini, asking for strict
JSON containing a recommended settlement percentage, suggested amount, monthly payment plan, and
reasoning. If Gemini is unavailable, times out, or returns unparseable output, a deterministic
affordability-based heuristic computes the same fields so the feature never hard-fails.

**Negotiation letters** (`generate_negotiation_letter`): asks Gemini to draft a lender-specific,
professional email proposing a reduced settlement. On failure, a fallback text template fills in
the lender name and amounts.

Every AI call (successful or fallback) is logged to `ai_history` for auditing, and flagged with
`used_fallback` so you can see when the deterministic path was used.

## Running Locally

### Backend
```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp ../.env.example .env   # then set JWT_SECRET_KEY and GEMINI_API_KEY
uvicorn app.main:app --reload
```
API docs: http://localhost:8000/docs

Seed demo data (from the `backend/` directory, after installing deps):
```bash
python ../scripts/init_db.py
# Demo login: demo@debtrelief.ai / DemoPass123
```

Run tests:
```bash
pytest -q
```

### Frontend
```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```
App: http://localhost:5173

```bash
cp .env.example .env   # fill in JWT_SECRET_KEY and GEMINI_API_KEY
docker compose up --build
```
- Frontend: http://localhost:5173
- Backend: http://localhost:8000/docs

## Production Considerations

- **Security:** Set a strong, unique `JWT_SECRET_KEY`; restrict `ALLOWED_ORIGINS` to your real
  frontend domain; serve behind HTTPS (e.g. via a reverse proxy/load balancer); all DB access goes
  through SQLAlchemy/SQLModel, avoiding raw SQL injection risk; add rate limiting (e.g. `slowapi`)
  on `/auth/*` and AI endpoints if exposed publicly.
- **Error handling:** Global exception handlers return sanitized JSON errors; validation errors
  return structured 422 responses.
- **Testing:** `backend/tests/` covers financial calculation logic and the full auth → loan →
  settlement → letter → history flow via FastAPI's `TestClient`.
- **Logging:** Python `logging` is configured in `app/main.py`; extend with structured logging
  (e.g. `structlog`) and a log shipper for production.
- **Config:** All secrets/config load via `pydantic-settings` from environment variables or a
  `.env` file — nothing is hard-coded.
- **Database:** Swap `DATABASE_URL` to a Postgres connection string to move off SQLite; introduce
  Alembic migrations (already in `requirements.txt`) instead of relying on `create_all` for schema
  changes in production.

## Notes on Scaffolding

This is a complete, runnable scaffold — not just snippets. The backend has been verified with
`pytest` (auth, loan CRUD, settlement generation, letter generation, and history all pass end to
end using the fallback AI logic, since no Gemini key is configured in this sandbox). To enable
live Gemini output, set `GEMINI_API_KEY` in `.env`.

The frontend is a complete Vite/React app; run `npm install && npm run dev` to try it against the
backend. `npm install` was not run in this sandbox (no need to vendor `node_modules` into the
deliverable), so do that first locally.
